Nextcloud Docker Setup Script

This interactive shell script helps you quickly deploy a Nextcloud instance inside a Docker container. It guides users through configuration via whiptail dialogs and supports optional Redis integration and custom data paths.

Step-by-Step Process

  1. Prompt for trusted_proxies to be set in the Nextcloud configuration.
  2. Ask whether to use a custom data directory path (e.g., /home/nextcloud) or use the default path (/opt/nextcloud).
  3. Optionally enable Redis caching by providing the Redis serverâ€™s IP and port.
  4. Select the default language (German or English); this also sets the default phone region accordingly (DE or US).
  5. Enter your Nextcloud domain (without https://), e.g., cloud.example.com.
  6. Create necessary folders (config, html, data) and set proper permissions for the www-data user.
  7. Generate a valid config.php file including all selected options.
  8. Launch the Nextcloud Docker container with mounted volumes and the specified configuration.
  9. Display a success message including the final URL to access the Nextcloud web interface.

Once completed, you'll have a fully operational Nextcloud container ready to be finalized via the web interface. The script supports optional Redis caching and flexible data directory locations for advanced setups.
