Docker Container Migration Script

This advanced interactive shell script allows you to migrate Docker containers from one host to another over SSH. It provides a user-friendly interface using whiptail and includes progress visualization for each migration step. The script handles images, networks, volumes, bind mounts, and container configurations automatically.

Step-by-Step Process

  1. Check for required dependencies (whiptail, sshpass, jq, pv) and install them if missing.
  2. Prompt the user for target server connection details, including IP address, SSH port, username, and password.
  3. Verify that Docker is installed on both the source and target systems.
  4. Display a checklist of all running Docker containers on the source system for selection.
  5. For each selected container, retrieve its configuration using docker inspect.
  6. Pull the required Docker image on the target system.
  7. Recreate all Docker networks used by the container on the target host if they do not already exist.
  8. Transfer all Docker volumes:
    • Automatically create missing volumes on the target.
    • Stream data securely using tar over SSH with progress tracking via pv.
  9. Transfer bind-mounted directories by copying their contents to the target system, preserving structure.
  10. Reconstruct container runtime configuration, including:
    • Port bindings
    • Environment variables
    • Restart policies
    • User settings
    • Startup command
  11. Start the container on the target system using the reconstructed configuration.
  12. Display a real-time progress bar for each container migration using whiptail.
  13. Repeat the process for all selected containers.
  14. Show a final success message once all migrations are completed.

After completion, the selected Docker containers will be fully migrated and running on the target system with their data and configurations preserved. This script is especially useful for server migrations, backups, or moving workloads between hosts with minimal manual intervention.
